Prim's Algorithm

A greedy algorithm that builds the Minimum Spanning Tree (MST) by starting from an arbitrary node and repeatedly adding the shortest edge that connects a node in the MST to a node outside it.

Key Steps:

  • Start with any node as the initial MST
  • Maintain a priority queue of edges connecting MST nodes to non-MST nodes
  • At each step, add the minimum-weight edge from the priority queue
  • Stop when all nodes are included in the MST

Algorithm Properties:

  • Time Complexity: O(E log V) using binary heaps
  • Space Complexity: O(V + E)
  • Optimality: Always produces a correct MST
  • Best For: Dense graphs where E ≈ V²
  • Data Structures: Priority Queue, Visited Set

Real-World Applications:

  • Network design (telecommunications, roads)
  • Cluster analysis
  • Image segmentation
  • Circuit design

Kruskal's Algorithm

A greedy algorithm that builds the MST by sorting all edges by weight and adding them in ascending order, skipping any edge that would form a cycle.

Key Steps:

  • Sort all edges in non-decreasing order of weight
  • Initialize a forest (each node as separate tree)
  • Iterate through sorted edges, adding each edge if it connects different trees
  • Stop when V-1 edges are added (complete MST)

Algorithm Properties:

  • Time Complexity: O(E log E) for sorting + O(E α(V)) for union-find
  • Space Complexity: O(V + E)
  • Optimality: Always produces a correct MST
  • Best For: Sparse graphs where E ≈ V
  • Data Structures: Union-Find (Disjoint Set), Sorted Edges

Real-World Applications:

  • LAN network design
  • Electric grid construction
  • Airline route planning
  • Water supply networks
